# SpaceGraphJS — Redemption Plan

**Status:** The codebase is real. TypeScript source exists in `src/`, builds via Vite, has test infrastructure (Vitest, Playwright), a vision system with analyzers and strategies, and a demo app with 30+ demos. The ONNX models are verified dummies (generated by `generate_dummy_odn.py` — 183 bytes for ODN, sub-5KB for all 5). The retro visual identity of the Java/C++ ancestors was lost in translation.

This plan fixes what matters: aesthetic identity, working vision without placeholder ML, and demo polish.

---

Before building, understand what we actually have.

- [ ] **Audit what builds and tests pass** — `pnpm run build:lib`, `pnpm test`, `pnpm run test:demo-errors`. Document failures
- [ ] **Audit the 5 ONNX models** — confirm they are dummies via Python `onnx.load()`. Document what outputs they produce vs what the spec claims

- [ ] **Check the demo page** — `dist-demo/index.html` works? All 30+ demos load without console errors?
- [ ] **Verify framework bindings** — what state are `packages/react-spacegraph`, `solid-spacegraph`, `vue-spacegraph` really in?
- [ ] **Optionally: capture legacy screenshots** — `spacegraphc` builds and runs under Xvfb. May be useful reference for the retro theme. Not blocking.

## Phase 1: Retro CSS Theme (Weeks 1-2)

The library renders generic Three.js. Give it the visual soul of the C++/Java versions — through CSS, not GLSL.

- [ ] **RetroTheme CSS preset** — dark background (`#000` or `#111`), monospace font, low-saturation gray palette, neon accent colors (cyan, amber, green)
- [ ] **Canvas filter overlay** — CSS `filter: contrast(1.05) saturate(0.8) brightness(0.9)` on the canvas container for a muted/CRT feel
- [ ] **Optional scanline overlay** — single CSS pseudo-element with `repeating-linear-gradient`. Zero cost when off
- [ ] **Panel styling** — semi-transparent dark panels (`rgba(0,0,0,0.75)`) with thin bright borders matching spacegraphc HUD aesthetic
- [ ] **Color palette CSS custom properties** — port spacegraph1's extended gray scale as design tokens
- [ ] **Cursor** — CSS `crosshair` by default; hexagonal crosshair via `clip-path` if desired
- [ ] **Font** — `font-family: monospace` or one `@font-face` retro bitmap font (e.g. Press Start 2P from Google Fonts). No Hershey parsing
- [ ] **Export as `applyRetroTheme(graph)`** — single API call that applies the theme. Demo pages should be one-liner toggles

## Phase 2: Working Vision — No ML Required (Weeks 2-3)

The ONNX models are tiny stubs. Replace them with real deterministic checks that produce actual useful output. The analyzer code is already well-structured (`OverlapAnalyzer.ts`, `LegibilityAnalyzer.ts`, etc. in `src/vision/analyzers/`).

- [ ] **Overlap detection** — `getBoundingClientRect()` intersection. Fast, exact, zero deps
- [ ] **WCAG contrast checking** — `(L1 + 0.05) / (L2 + 0.05)` on computed styles. Exact
- [ ] **Font size legibility** — `getComputedStyle()` pixel measurement vs threshold
- [ ] **Color harmony** — Itten complementary/analogous/triadic rules. Deterministic
- [ ] **Ergonomics** — Fitts's Law from element sizes and positions
- [ ] **Wire into HeuristicsStrategy** — the heuristics path likely already exists (`src/vision/strategies/`). Ensure it's the default. Move ONNX/dummy model loading behind an optional flag
- [ ] **Vision panel** — collapsible overlay showing live scores. Styled with retro theme (dark bg, monospace, thin borders)
- [ ] **Drop onnxruntime-web from critical path** — make it an optional dynamic import, not a dependency that blocks vision

## Principles

1. **Retro feel via CSS, not shaders.** Color, font, spacing, overlays. No GLSL porting.
2. **CSS/DOM analysis > placeholder ONNX.** Working simple code beats non-working ML every time.
3. **Tests must pass.** Green CI is non-negotiable.
4. **The library already works.** Most of the code is written. Finish it, don't rebuild it.
